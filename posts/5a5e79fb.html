Convex Hull and Sorting

convex hull algorithms: O(nlog⁡n)O(n\log n)O(nlogn).

can we do better?

convex hull of points is equivalent to sorted numbers.

O(nlog⁡n)O(n\log n)O(nlogn) worst-case is optimal.

fewer than n points on the hull

O(nlog⁡h)O(n\log h)O(nlogh)?

Marriage Before Conquest

MbC(P:l,r): find Upper Hull of P from x-coordinates l to r

  1. find median x-coordinate xmx_mxm​. partition into Pl,PrP_l,P_rPl​,Pr​. O(n)O(n)O(n) times
  2. find the bridge, uv, over xmx_mxm​.
  3. remove the points below the bridge. O(n)O(n)O(n)
  4. recurse: MbC(Pl:l,ux)MbC(P_l:l,u_x)MbC(Pl​:l,ux​) & MbC(Pr:vx,r)MbC(P_r:v_x,r)MbC(Pr​:vx​,r) f(n2)+f(n2)f(\frac{n}{2})+f(\frac{n}{2})f(2n​)+f(2n​)

if step 2 can be done in linear time, the total time is O(nlog⁡h)O(n\log h)O(nlogh).

How to find bridges

finding a bridge is equivalent to finding a line Y=aX+bY=aX+bY=aX+b that:

  • passes above all the points
  • has the lowest intersection point with the separating line (the lowest point that is not inside the convex hull)

find aaa and bbb s.t.:

  • yi≤axi+by_i\le ax_i+byi​≤axi​+b for every input point (xi,yi)(x_i,y_i)(xi​,yi​)
  • ym=axm+by_m=ax_m+bym​=axm​+b is minimized

1D Linear Programming

unknown x1x_1x1​

minimize c1x1c_1x_1c1​x1​ s.t. ai,1x1≤bia_{i,1}x_1\le b_iai,1​x1​≤bi​ for i in 1 to n

possible cases: feasible, infeasible or unbounded.

O(n)O(n)O(n) running time

2D Linear Programming

unknown x1,x2x_1,x_2x1​,x2​

minimize c1x1+c2x2c_1x_1+c_2x_2c1​x1​+c2​x2​ s.t. ai,1x1+ai,2x2≤bia_{i,1}x_1+a_{i,2}x_2\le b_iai,1​x1​+ai,2​x2​≤bi​ for i in 1 to n

假设一些直线找到了一个可行区域

现在就只需要平移c1x1+c2x2=Vc_1x_1+c_2x_2=Vc1​x1​+c2​x2​=V 这条直线，以找到最小值。

  1. initialize vvv.
  2. for i=1 to n
    1. if constraint i does not violate v: do nothing
    2. else solve 1D Linear programming with constraints 1 to i-1 on boundary of constraint i

step 2.2 might run n times O(n2)O(n^2)O(n2) time in total.

A randomized incremental algorithm

  1. initialize v
  2. randomly relabel (shuffle) constraints
  3. for i = 1 to n

    1. if constraint i does not violate v: do nothing

    2. else solve 1D Linear Programming with constraints 1 to i-1 on boundary of constraint i

      意思就是新增的这个约束 i 对应的直线，把 x2 解出来，代入到前面 1 到 i-1 中的 x2，可以得到一个一维的线性规划问题。

improves from O(n2)O(n^2)%O(n2) to O(n)O(n)O(n) time on average.

backward analysis (a claim): the probability that step 3.2 runs during the i-th iteration of the for loop is ≤2i\le\frac{2}{i}≤i2​

expected running time:

how to shuffle?

Place i in a random non-empty spot for i = 1 to n

fill i-th spot randomly for i = 1 to n

fill i-th spot randomly from i = n to 1

To proof the claim:

consider the step 3.2

every constraint is a line, we have n of them. c1,⋯ ,cnc_1,\cdots,c_nc1​,⋯,cn​.

we have n positions in our shuffle.

the line is placed randomly in these n positions.

then the probability of entering 3.2:

  • Probability entering 3.2 at i=ni=ni=n is: 2n\frac{2}{n}n2​, because only two lines determin final result. since the order of lines are shuffled, the probability of n-th line is one of the two determin line is 2n\frac{2}{n}n2​.
  • consider feasible region, the optimal solution is adjacent to 2 constraints c1,c2c_1,c_2c1​,c2​. for i-th line probability would be 2i\frac{2}{i}i2​

Chan’s Algortihm

陈氏算法，计算凸包的一种快速算法，时间复杂度 O(nlog⁡h)O(n\log h)O(nlogh)

预设一个凸包上的点的个数 h≤22ih\le2^{2^i}h≤22i. iii 从 111 开始尝试。

对每一个 i：

  • 将当前所有点分成 nh\frac{n}{h}hn​ 份，分别用 graham scan 计算凸包，这一步的时间复杂度为 O(hlog⁡h)O(h\log h)O(hlogh)。计算 nh\frac{n}{h}hn​ 次，所以总的时间复杂度为 O(nlog⁡h)O(n\log h)O(nlogh)。

  • 从这些凸包上 x 坐标最小的点开始，找到这 nh\frac{n}{h}hn​ 个凸包合并的上凸包。就是对当前的点，找到通过分别二分查找 O(log⁡h)O(\log h)O(logh) 这 nh\frac{n}{h}hn​ 个子凸包上和当前点连线正切值最小的点。这一步是 O(nhlog⁡h)O(\frac{n}{h}\log h)O(hn​logh).

    因为假设凸包点数的上界是 h，最多要找 h 次，所以找到这些子凸包的公共上凸包的总时间为 O(nlog⁡h)O(n\log h)O(nlogh)

    如果假设正确，那么整个算法的时间复杂度为 O(nlog⁡h)O(n\log h)O(nlogh)。

  • 如果算法错误，i++，并继续尝试计算。

    假设最终凸包上的点数为 hhh，那么最多测试 log⁡log⁡h\log\log hloglogh 次。

    所以总的时间复杂度为
