Fair Division for 2 Strategic Agents with Divisible Items

Created2024-04-27|Updated2024-04-27|AUDatalogi算法博弈论公平分配博弈
|Word Count:1.4k|Reading Time:8mins|Post Views:

For randomized fair division with indivisible items, we consider its ex-ante fairness. Indeed when it comes to expected valuation (or payoff), it’s equivalent to fair division with divisible items.

Consider fair division with divisible items among only 2 AGENT.

The input valuation for each agent is the ordinal preference ≻i\succ_i≻i​.

We want to explore to what extent agent can manipulate by misreporting preference profile to gain more share of items with higher priority.

The source code and experiment results is available in my GitHub Repo

Reporting State Graph

A (directed) reporting state graph is defined: G=(V,E)G=(V,E)G=(V,E).

VVV denote the set of all the possible preference input.

EEE denotes the set of all edges. A directed edge e=(v1,v2)e=(v_1,v_2)e=(v1​,v2​) denotes there exist a agent in reporting state v1v_1v1​, who can better off by deviating its report to state v2v_2v2​.

(In this scenario, better off means that agent strictly SD-prefer v2v_2v2​ to v1v_1v1​)

  • For case of 3 items, there are 3!×3!=363!\times3!=363!×3!=36 reporting state vertices.
  • For case of 4 items, there are 4!×4!=5764!\times4!=5764!×4!=576 reporting state vertices.

❓Is there any cycle in the reporting state graph?

# States Can be Reduced

Without loss of generality, by rotational symmetry,

  • For case of 3 items, it suffices to analyze only 3!=63!=63!=6 reporting state vertices.

    For example 1:g1≻g2≻g3,2:g1≻g2≻g31:g_1\succ g_2\succ g_3,2:g_1\succ g_2\succ g_31:g1​≻g2​≻g3​,2:g1​≻g2​≻g3​ is equivalent to 1:g1≻g3≻g2,2:g1≻g3≻g21:g_1\succ g_3\succ g_2,2:g_1\succ g_3\succ g_21:g1​≻g3​≻g2​,2:g1​≻g3​≻g2​.

  • Analogously, for case of 444 items, it’s enough to analyze 4!=244!=244!=24 reporting states.

2 Agents, 3 Items

Without loss of generality, by symmetry of two agents, it suffice to check only one agent.

Analysis Case-by-case

For the scenario of 2 agents with 3 divisible goods.

Assume the 3 items are g1,g2,g3g_1,g_2,g_3g1​,g2​,g3​, 2 agent 1,21,21,2.

  • There are 363636 cases in total, but by rotational symmetry, it can be reduced to 666 cases.
  • For example 1:g1≻g2≻g3,2:g1≻g2≻g31:g_1\succ g_2\succ g_3,2:g_1\succ g_2\succ g_31:g1​≻g2​≻g3​,2:g1​≻g2​≻g3​ is equivalent to 1:g1≻g3≻g2,2:g1≻g3≻g21:g_1\succ g_3\succ g_2,2:g_1\succ g_3\succ g_21:g1​≻g3​≻g2​,2:g1​≻g3​≻g2​.
  • The 666 cases are in the following (Fix the ordinal preference on agent 1 on g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​, we analysis agent 2 for all permutations on preference):

  1. agent 1’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent 2’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent 1 gets: [50.0,50.0,50.0][50.0, 50.0, 50.0][50.0,50.0,50.0]

    agent 2 gets: [50.0,50.0,50.0][50.0, 50.0, 50.0][50.0,50.0,50.0]

    • Best response for agent 2? Yes.

      Because there are no other preference profile of player 2 inducing high sum of share of first kkk item for k=[3]k=[3]k=[3].

  2. agent1’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent2’s true preference: g1≻g3≻g2g_1\succ g_3\succ g_2g1​≻g3​≻g2​

    agent1 gets: [50.0,100,0][50.0, 100, 0][50.0,100,0]

    agent2 gets: [50.0,0,100][50.0, 0, 100][50.0,0,100]

    • Best response for agent 2? Yes.

      First observe that by eating protocol, each agent exactly gain the total share of 150150150 item.

      The top priority of both agents are g1g_1g1​, thus agent 2 can’t manipulate to gain higher than 505050 share of g1g_1g1​ when agent 1 reports truthfully by putting g1g_1g1​ on its top priority.

    • Agent 2 in case 5 has an incentive to deviate to case 2. Then it’s natural to ask whether it’s the best response for agent 1.

      The answer is Yes.

      • Agent 1 must put g1g_1g1​ on top priority, otherwise it will loss the share of its favorite g1g_1g1​.
      • If agent 111 switch the position of g2g_2g2​ and g3g_3g3​, it will loss the share of second priority g2g_2g2​.

  3. agent 1’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent 2’s true preference: g2≻g1≻g3g_2\succ g_1\succ g_3g2​≻g1​≻g3​

    agent 1 gets: [100,0,50.0][100, 0, 50.0][100,0,50.0]

    agent 2 gets: [0,100,50.0][0, 100, 50.0][0,100,50.0]

    • Best response for agent 2? Yes.

      Here any other misreport which promote share of g1g_1g1​ will hurt the 100100100 share of g2g_2g2​.

  4. agent 1’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent 2’s true preference: g2≻g3≻g1g_2\succ g_3\succ g_1g2​≻g3​≻g1​

    agent1 gets: [100,0,50.0][100, 0, 50.0][100,0,50.0]

    agent 2 gets: [0,100,50.0][0, 100, 50.0][0,100,50.0]

    • Best response for agent 2? Yes.

      Indeed, it’s the best outcome for both agent: 100100100 share of top priority, 505050 share of second priority.

  5. agent 1’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent 2’s true preference: g3≻g1≻g2g_3\succ g_1\succ g_2g3​≻g1​≻g2​

    agent1 gets: [100,50.0,0][100, 50.0, 0][100,50.0,0]

    agent2 gets: [0,50.0,100][0, 50.0, 100][0,50.0,100]

    • Best response for agent 2? NO!!!

      By misreporting g1≻g3≻g2g_1\succ g_3\succ g_2g1​≻g3​≻g2​, the eating protocol allocates:

      • agent 1 gets: [50.0,100,0][50.0, 100, 0][50.0,100,0]
      • agent 2 gets: [50.0,0,100][50.0, 0, 100][50.0,0,100]

      Which is strictly better off w.r.t. agent 2.

  6. agent 1’s true preference: g1≻g2≻g3g_1\succ g_2\succ g_3g1​≻g2​≻g3​

    agent 2’s true preference: g3≻g2≻g1g_3\succ g_2\succ g_1g3​≻g2​≻g1​

    agent 1 gets: [100,50.0,0][100, 50.0, 0][100,50.0,0]

    agent 2 gets: [0,50.0,100][0, 50.0, 100][0,50.0,100]

    • Best response for agent 222? Yes.

      Indeed, it’s the best outcome for both agent: 100100100 share of top priority, 505050 share of second priority.

Graph of reporting states

There are 3!×3!=363!\times3!=363!×3!=36 reporting states VVV. An directed edge in the graph e=(v1,v2)∈Ee=(v_1,v_2)\in Ee=(v1​,v2​)∈E means there are a agent in reporting state node v1v_1v1​ has an incentive to deviate to reporting state v2v_2v2​.

By the analysis for 6 cases above, there are no cycle in the graph.
